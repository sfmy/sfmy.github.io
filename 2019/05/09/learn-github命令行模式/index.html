    命令行更有利于让我们知道究竟发生了什么

    查看帮助

    1.git help
    查看所有git命令
    2.git help 命令
    查看某个命令的详细帮助

    克隆命令

    1.git clone ‘https://github.com/sfmy/learn-github.git'
    从远程克隆仓库到本地
    2.git clone ‘https://github.com/sfmy/learn-github.git' learn
    从远程克隆仓库到本地，并修改仓库名为learn

    拉取最新代码

    1.git pull
    从远端拉取最新代码

    查看状态

    1.git status
    查看当前工作状态
    1.git status – ‘目录’
    查看此目录的工作状态

    提交代码

    1.git add .
    提交当前工作空间的修改到缓存
    2.git add – ‘目录’
    提交目录的修改到缓存
    3.git commit -m ‘注释’
    提交缓存到本地(只有push才会推到远端)

    推到远程

    1.git push
    推送本地commit的内容到到远程

    查看变动

    1.git diff
    查看工作空间的文件变动内容
    2.git diff –cached
    查看提交到缓存的变动内容
    3.git diff HEAD
    查看工作空间的文件变动内容
    4.git diff – ‘路径’
    查看路径内的变动

    查看历史记录

    1.git log
    历史记录
    2.git log -p
    历史记录详细变动
    3.git log –author=sfmy
    提交者为sfmy的记录
    4.git log -3
    查看3次提交记录
    5.git log –stat
    查看记录的变动文件
    6.git log – ‘路径’
    查看路径内的记录
    7.git log –since yesterday
    查看从昨天开始的记录

    切换分支

    1.git checkout -b ‘分支名字’
    从当前分支clone一个分支出去
    2.git branch
    查看当前分支
    3.git branch -a
    查看本地所有分支
    4.git branch -r
    查看远端分支

    合并分支

    1.git merge ‘分支名字’
    合并分支到当前分支

    git好用的地方在哪里

    工作中，经常会有这种情况，正在开发一个功能，这个功能可能需要一周才能完成，你可以切出来一个功能分支，每完成一个小功能，提交一次，随时都可以查看自己的历史改动与工作空间的改动。并且在有其他bug时，还可以再切换到主分支上修改bug。但是使用svn的话，整个功能不完成的话，代码是不能提交的，这样当你第三天想看下今天都改动了什么，就只能查看第一天到第三天的改动。并且当有其他bug出现，而且这个需要改动你正在开发的功能已经改动的文件，此时你修改bug提交文件就会涉及前三天写的改动，但是你只是想提交修改bug的改动，而这个改动是会跟功能在同一个文件中。这就不好解决了。

    一些其它不常用的命令

    可以使用正则表达式进行某些匹配

    1.git log -3 -p – ‘.js’
    查看js文件最近3次的详细改动
    2.git add – ‘    .js’
    只提交js文件到缓存
    3.git commit – ‘*.js’ -m ‘xxx’
    提交js文件

    显示某次log的内容

    1.git show ‘log的index’

    暂存工作空间的改动

    当你正在开发功能分支时，需要切到其他分支修改bug，此时小功能还完成，不想提交，可以暂时把此分支未提交的改动存储起来。等待在其他分支修改完bug后再从stash中弹出来。
    1.git stash
    必须先’git add .’保存到缓存中，才能再git stash
    2.git stash save -m ‘注释’
    使用注释的话更方便提醒自己，缓存内容，防止出错
    3.git stash pop
    会使用最近的缓存内容，从stash中删除
    4.git stash apply stash@{index}
    使用缓存
    5.git stash drop stash@{index}
    删除缓存

    回退自己的修改

    1.git reset –soft HEAD^
    会把上次的提交重新回退到本地的缓存中,^个数为回退的次数

    修改本地的提交注释

    1.git rebase -i HEAD~n
    修改最近n次提交的注释

    更换电子邮件与用户名

    1.只在本项目修改名字与邮件
    git config –local user.name ‘your name’
    git config –local user.email ‘your email’
    2.全局修改名字与邮件
    git config –global user.name ‘your name’
    git config –global user.email ‘your email’

    git中文文件乱码

    git config –global core.quotepath false
